Core flow

The client submits a domain. The control plane does four things.

  • 01 Read domain, port, and exposure from the manifest.
  • 02 Write status.example.com -> cn-edge in Cloudflare.
  • 03 Ask Traefik to issue HTTPS for status.example.com.
  • 04 Update the stack so the Host route reaches status:80.

Quick start

Create the control endpoint, then deploy named services from clients.

Install the CLI

Install the same luma command on managers, workers, or clients.

curl -fsSL https://raw.githubusercontent.com/LiuTianjie/luma/main/scripts/install-luma.sh | sh
luma preflight

Create the control domain

Run this on the first full server to create the control endpoint and deploy token.

luma bootstrap manager --domain luma.example.com --profile single-node

Add workload nodes as needed

Workers receive region and profile labels so the control plane can choose placement.

luma node join https://luma.example.com --token <join-token> --profile global-worker --region global

Deploy a named domain from a client

The control plane handles DNS, certificate issuance, routing, and workload updates.

luma login https://luma.example.com --token <deploy-token>
luma deploy status.yaml

Ready to publish a service?

Write the domain and port. Luma publishes it as HTTPS.
