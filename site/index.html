The operating model

A full manager, local workers, lightweight clients.

Bootstrap once on the manager. Join every worker from its own shell. After that, any laptop or throwaway box can log in and deploy without joining the cluster.

Manager Traefik + Portainer + Luma Control
Worker Swarm workload node
Client login + deploy only

Animated setup guide

Four commands, each with a visible outcome.

Install the CLI

Download a GitHub archive, create a private venv, and write a `luma` command shim.

curl -fsSL https://raw.githubusercontent.com/LiuTianjie/luma/main/scripts/install-luma.sh | sh
luma preflight

Bootstrap the manager

Run this on the first full server. It installs Docker, Swarm, Traefik, Portainer, Luma Control, and optional egress.

luma bootstrap manager --domain luma.example.com --profile single-node

Join worker nodes locally

Every worker asks Luma Control for the Swarm token, joins, then receives region and profile labels automatically.

luma node join https://luma.example.com --token <join-token> --profile global-worker --region global

Deploy from any client

The client sends a service manifest to Luma Control. The manager creates the domain route, syncs DNS, issues HTTPS, and updates Portainer.

luma login https://luma.example.com --token <deploy-token>
luma deploy status.yaml
status.yaml 
name: status
image: traefik/whoami:latest
region: cn
exposure: cn-edge
domain: status.example.com
port: 80
replicas: 2

What Luma handles

A tiny service file becomes a public HTTPS endpoint.

  • 01 Render Swarm stack and Traefik route for `status.example.com`.
  • 02 Sync Cloudflare DNS to the selected edge node.
  • 03 Pull `traefik/whoami` through egress, then fallback to mirrors if needed.
  • 04 Create or update the stack through Portainer with visible logs.

Roles

One product path, multiple node shapes.

A

Manager

Runs Swarm manager, Traefik, Portainer, Luma Control, tokens, DNS sync, and optional egress.

B/C

Workers

Join from their own shell. No repeat login required after labels are applied.

D

Clients

Do not join the cluster. Store only a login context and send manifests to the control API.
