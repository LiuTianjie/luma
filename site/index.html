Core flow

One domain field expands into DNS, TLS, routing, and service placement.

  • 01
    Parse the service entry

    Read domain, port, and exposure to resolve the entrypoint and target port.

  • 02
    Sync DNS record

    Write the Cloudflare record so the domain points at the selected edge node.

  • 03
    Create the HTTPS entry

    Traefik completes ACME from the Host rule and creates the TLS route.

  • 04
    Update service routing

    Update the stack so status.example.com forwards to status:80 replicas.

Quick start

Three machine roles. Three command paths.

Install the CLI

Managers, workers, and clients use the same luma command.

curl -fsSL https://raw.githubusercontent.com/LiuTianjie/luma/main/scripts/install-luma.sh | sh
luma preflight

Initialize the control plane

Create the Swarm manager, Traefik, Portainer, and Luma Control on the first full node.

luma bootstrap manager --domain luma.example.com --profile single-node

Join workload nodes

Each server joins locally, then the control plane records its region label and optional egress capability.

luma node join https://luma.example.com --token <join-token> --region global --name global-sg-1

Deploy from a client

After login, the client submits a manifest. The control plane handles DNS, TLS, routing, and stack updates.

luma login https://luma.example.com --token <deploy-token>
luma deploy status.yaml

Start validating

Deploy a status service first and verify the full entry path.
