自托管部署控制面

客户端声明域名，Luma 自动发布 HTTPS 服务。

在任意客户端执行 luma deploy，声明服务域名和端口。Luma 控制面会修改 DNS、 申请 HTTPS 证书、配置 Traefik 路由，并把流量转发到 Swarm 工作负载。

核心链路

从一个 domain 字段，到可访问的 HTTPS 服务。

  • 01 读取 manifest 里的 domain、port、exposure。
  • 02 在 Cloudflare 写入 status.example.com -> cn-edge。
  • 03 让 Traefik 为 status.example.com 自动签发 HTTPS 证书。
  • 04 更新 stack，把 Host 路由转发到 status:80 工作负载。

上手路径

先建立控制面，再从客户端部署域名服务。

安装 CLI

在 manager、worker 或客户端上都可以安装同一个 luma 命令。

curl -fsSL https://raw.githubusercontent.com/LiuTianjie/luma/main/scripts/install-luma.sh | sh
luma preflight

初始化控制面域名

在第一台全功能服务器上执行，创建控制面入口和部署 token。

luma bootstrap manager --domain luma.example.com --profile single-node

按需加入工作节点

worker 加入后获得 region 和 profile labels，后续由控制面选择落点。

luma node join https://luma.example.com --token <join-token> --profile global-worker --region global

客户端部署指定域名

客户端提交 manifest 后，控制面自动完成 DNS、证书、路由和工作负载更新。

luma login https://luma.example.com --token <deploy-token>
luma deploy status.yaml

准备发布服务？

写下域名和端口，Luma 自动发布成 HTTPS 服务。
