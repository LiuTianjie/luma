自托管部署控制面

在任何机器上，把服务部署到自己的集群。

Luma 把分散的服务器组织成一个可登录的 Swarm 集群。每个服务都有真实域名、 自动 DNS、HTTPS 证书、Portainer 可视化，以及能感知代理环境的镜像拉取路径。

运行模型

一个全功能 manager，本机加入 workers，轻量 clients。

先在 manager 上完成 bootstrap。每台 worker 都在自己的 shell 里加入集群。 之后任何笔记本或临时机器都只需要 login，就能部署到集群。

Manager Traefik + Portainer + Luma Control
Worker Swarm 工作负载节点
Client 只负责 login + deploy

带动效的安装路径

四条命令，每一步都有可见结果。

安装 CLI

下载 GitHub archive，创建用户级 venv，并写入 `luma` 命令入口。

curl -fsSL https://raw.githubusercontent.com/LiuTianjie/luma/main/scripts/install-luma.sh | sh
luma preflight

初始化 manager

在第一台全功能服务器上执行。它会安装 Docker、Swarm、Traefik、Portainer、Luma Control，以及可选 egress。

luma bootstrap manager --domain luma.example.com --profile single-node

worker 本机加入

每台 worker 向 Luma Control 请求 Swarm token，加入集群后自动获得 region 和 profile labels。

luma node join https://luma.example.com --token <join-token> --profile global-worker --region global

从任意 client 部署

client 把服务清单提交给 Luma Control。manager 负责创建域名路由、同步 DNS、签发 HTTPS，并更新 Portainer。

luma login https://luma.example.com --token <deploy-token>
luma deploy status.yaml
status.yaml 
name: status
image: traefik/whoami:latest
region: cn
exposure: cn-edge
domain: status.example.com
port: 80
replicas: 2

Luma 负责什么

一份小服务文件，展开成公网 HTTPS 入口。

  • 01 为 `status.example.com` 渲染 Swarm stack 和 Traefik 路由。
  • 02 把 Cloudflare DNS 同步到选中的 edge 节点。
  • 03 先通过 egress 拉 `traefik/whoami`，失败后再切换镜像源兜底。
  • 04 通过 Portainer 创建或更新 stack，并保留可见日志。

角色

一条产品路径，适配多种节点。

A

Manager

运行 Swarm manager、Traefik、Portainer、Luma Control、token、DNS 同步和可选 egress。

B/C

Workers

在各自服务器上本机加入。labels 应用后，后续部署不需要反复登录每台机器。

D

Clients

不加入集群。只保存登录上下文，并把服务清单发送给控制面 API。

准备组装集群？

安装 Luma，绑定域名，把服务文件变成 HTTPS 入口。
