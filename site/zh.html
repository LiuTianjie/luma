核心链路

一个 domain 字段，展开成 DNS、证书、路由和服务。

  • 01
    解析服务入口

    读取 domain、port 和 exposure，确定入口与目标端口。

  • 02
    同步 DNS 记录

    在 Cloudflare 写入记录，让域名指向选中的边缘节点。

  • 03
    建立 HTTPS 入口

    Traefik 根据 Host 规则完成 ACME，并生成对应的 TLS 路由。

  • 04
    更新服务路由

    更新 stack，把 status.example.com 转发到 status:80 副本。

上手路径

三类机器，对应三段命令。

安装 CLI

manager、worker、client 使用同一个 luma 命令。

curl -fsSL https://raw.githubusercontent.com/LiuTianjie/luma/main/scripts/install-luma.sh | sh
luma preflight

初始化控制面

创建唯一控制面 manager，同时启动 Swarm、Traefik、Portainer、Luma Control 和 egress gateway。

luma bootstrap manager --domain luma.example.com

加入工作节点

每台服务器本机 join 并写入 region 标签。name 只是机器标识，region 才是调度边界。

luma node join https://luma.example.com --token <join-token> --region global --name global-sg-1

从客户端部署服务

client 登录后提交 manifest，并在控制面处理 DNS、证书、路由、镜像拉取和 stack 更新时打印进度。

luma login https://luma.example.com --token <deploy-token>
luma deploy status.yaml

开始验证

先发布一个 status 服务，确认整条入口链路。
